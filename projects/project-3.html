Design System & Tokens

Scaling design consistency across 5 products

Role Design Systems Lead
Timeline 9 Months
Team 12 People

The Problem

As the company scaled, we had 5 different products being designed and developed in parallel by separate teams. Each team had made their own design decisions, resulting in inconsistent UI patterns, colors, typography, and component behavior. This fragmented experience hurt brand cohesion and slowed down design-to-development handoff.

Developers were rebuilding the same components across products, and designers were recreating wheels instead of focusing on solving user problems. We needed a single source of truth.

Goals & Constraints

Primary Goal

Create a unified design system used by all 5 products

Secondary Goal

Reduce design-to-dev handoff time by 50%

Constraint 1

Work doesn't block existing product development

Constraint 2

Support multiple platforms (Web, iOS, Android)

Design Process

Audit & Strategy

First, I conducted a comprehensive design audit across all 5 products, documenting:

  • Color palettes and usage patterns
  • Typography implementations
  • Existing components and patterns
  • Breaking points and inconsistencies

Foundation Building

We established a design language with:

  • Color system: Semantic tokens with light/dark mode support
  • Typography: Consistent scale and hierarchy across platforms
  • Spacing: 8px grid-based system
  • Icons: Curated icon library with consistent stroke weight

Component Library

Built 30+ reusable components with documentation:

  • Buttons, forms, modals, navigation components
  • Data display (tables, charts, lists)
  • States documented for each component

Implementation & Documentation

Created comprehensive documentation in Figma, including design tokens in code (JSON/CSS variables), accessibility guidelines, and usage examples. Built a component library in React for Web products.

Solution

Design Tokens

Created a complete token system (colors, typography, spacing, shadows, borders) that lives in code and design tool simultaneously. Tokens are semantic and platform-agnostic, enabling consistency across Web, iOS, and Android.

Component Library

Built a React component library that implements the design system in code. Every component includes accessible HTML, keyboard navigation, and dark mode support. Component documentation includes usage guidelines and design rationale.

Figma Design Kit

Created Figma library with all components, styles, and tokens. Designers can now create mockups that automatically reflect design system changes. Components are organized logically and clearly documented.

Guidelines & Governance

Established design system governance with clear processes for proposing new components, deprecating old patterns, and handling exceptions. Created quarterly review process to keep the system current.

Tangible Impact

Developers reported spending 40% less time on UI implementation because components were pre-built. Designers could prototype 50% faster using the component library. Brand consistency improved dramatically with 95% of UI following the design system on the first pass.

Key Learnings

  • Design systems work best when driven by real product needs, not theory
  • Cross-functional buy-in is critical—involve dev and design teams from the start
  • Living documentation is essential; static docs become outdated
  • Semantic tokens allow flexibility for dark mode and theming without duplication
  • Governance must balance consistency with the need to innovate
